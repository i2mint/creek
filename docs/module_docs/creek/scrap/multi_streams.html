creek.scrap.multi_streams¶

class creek.scrap.multi_streams.MergedStreams(streams_map: Mapping[Any, Iterable], sort_key: Optional[Callable] = None)[source]¶

Creates an iterable of (stream_id, stream_item) pairs from a {stream_id: stream, …} map. The stream_item will be yield in sorted order. Sort behavior can be modified by the sort_key argument which behaves like key` arguments of built-in like sorted, heapq.merge, itertools.groupby, etc. If given, the sort_key function applies to stream_item (not to stream_id).

>>> streams_map = {
...     'hello': [(2, 'two'), (3, 'three'), (5, 'five')],
...     'world': [(0, 'zero'), (1, 'one'), (3, 'three'), (6, 'six')]
... }
>>> streams_items = MergedStreams(streams_map)
>>> it = iter(streams_items)
>>> list(it)  
[('world', (0, 'zero')),
 ('world', (1, 'one')),
 ('hello', (2, 'two')),
 ('hello', (3, 'three')),
 ('world', (3, 'three')),
 ('hello', (5, 'five')),
 ('world', (6, 'six'))]
class creek.scrap.multi_streams.Pipe(*funcs)[source]¶

Simple function composition. That is, gives you a callable that implements input -> f_1 -> … -> f_n -> output.

>>> def foo(a, b=2):
...     return a + b
>>> f = Pipe(foo, lambda x: print(f"x: {x}"))
>>> f(3)
x: 5

Notes

  • Pipe instances don’t have a __name__ etc. So some expectations of normal functions are not met.

  • Pipe instance are pickalable (as long as the functions that compose them are)

creek.scrap.multi_streams.multi_stream_items(streams_map: Mapping[Any, Iterable])[source]¶

Provides a iterable of (k1, v1_1), (k1, v1_2), … >>> streams_map = {‘hello’: ‘abc’, ‘world’: [1, 2]} >>> hello_items, world_items = multi_stream_items(streams_map) >>> list(hello_items) [(‘hello’, ‘a’), (‘hello’, ‘b’), (‘hello’, ‘c’)] >>> list(world_items) [(‘world’, 1), (‘world’, 2)]
